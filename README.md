# photo-genka

**X-E5 の写真1枚あたりの値段が見れるだけの WEB アプリ** — https://x-e5.ojimpo.com

```
レンズキットの支払総額 ÷ カメラ内シャッターカウンタ = いま、1枚あたりいくらか
```

撮れば撮るほど1枚単価が下がっていく（元が取れていく）様子を、ただ眺めて楽しむためのページです。

## 背景・動機

FUJIFILM X-E5 のレンズキットを SMBC ショッピングクレジットの48回無金利分割で買いました。¥248,310。
決して安い買い物ではないので、「使い倒して元を取る」を数字で見たくなりました。

カメラの原価償却が写真の原価になる構造 — リポジトリ名の genka は「原価」と「減価」を二重に受けています。

初日の1枚目を撮った時点で、この写真の原価は ¥248,310/枚。ここから撮るたびに単価が双曲線を描いて下がっていき、いつかフィルム時代の1枚単価（フィルム代＋現像代 ÷ 24枚）を下回る日が来ます。その損益分岐点を眺めるアプリです。

## 設計思想

### 「この一台のためのページ」

汎用の写真単価計算アプリではありません。URL が機体名（`x-e5.ojimpo.com`）なのはそのためです。
将来別のボディを買ったら、同じエンジンを別の機体名サブドメインでデプロイする拡張パターンを想定しています。

### 分母はカメラ内カウンタ（ImageCount）

撮影枚数は写真管理サービスのアセット数ではなく、**Fujifilm の Maker Notes タグ 0x1438（ImageCount）** を使います。これはカメラ本体が撮って出し JPEG/HEIF/RAF に毎回埋め込むカウンタで、取り込み漏れやカメラ内削除の影響を受けません。Immich のアセット数は乖離チェック用の副次情報です。

ImageCount は 16bit なので 32,768 でロールオーバーします。前回スナップショットより小さい値が来たら +32,768 して単調増加に補正する処理を初期から入れています（そこまで撮るのが目標でもある）。

### 経年変化の2軸

トップの X-E5 イラストは時間と枚数で独立に変化します。

- **キラキラ（新品感）** = 時間ベース。購入から1年かけて消えていく
- **アタリ（塗装剥げ）** = 枚数ベース。マイルストーン到達で段階的に現れる

「古びる」のではなく「相棒になっていく」演出です。Fujifilm 文化では塗装剥げは勲章なので。
アタリの閾値は自分の iPhone の年間撮影実績（Immich 実測 約3,000枚/年）から較正し、半年・1年・2年・4年・10年相当に置いています。

## 仕組み

```
Immich（写真サーバー）
   │  ① 最新の X-E5 アセットを検索（/api/search/metadata, model=X-E5）
   │  ② オリジナルをダウンロード
   ▼
exiftool で ImageCount を読む → SQLite に日次スナップショット（JST 23:30）
   ▼
FastAPI が /api/stats /api/history を配信 → 素の HTML/CSS/JS が描画
```

- 初回起動時は購入日から今日まで、各日最後の1枚をサンプリングして遡ってバックフィル
- 写真をドロップすると、その写真の ImageCount から「何枚目・撮影時点の単価・現在の単価」を表示（画像は即時破棄、保存しません）
- 撮影枚数に応じた雑学マイルストーン（トランプ1組、千羽鶴、年間ティッシュ消費…）を閾値テーブルで切り替え
- 48回分割の支払進捗バー付き。ただし**演出専用**で、単価の分子は総額固定のまま

## 動かし方

```bash
cp .env.example .env   # 金額・購入日・Immich API キーを設定
docker compose up -d --build
```

Immich の API キーは `asset.read` / `asset.download`（＋乖離チェックを使うなら `asset.statistics`）の最小権限で発行します。

## 構成

- backend: FastAPI + SQLite + exiftool（`python:3.12-slim` に `libimage-exiftool-perl`）
- frontend: 素の HTML/CSS/JS。ビルドなし。デザインは [docs/mockups/](docs/mockups/) のモックが正
- 公開: Cloudflare Tunnel で `x-e5.ojimpo.com` → `localhost:8403`
